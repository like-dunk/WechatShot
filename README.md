# 视频号/头条号/抖音/小红书批量截图 Chrome 插件版

这是独立的 Chrome Manifest V3 插件版本，不依赖现有 Python/PyInstaller 程序，也不会改动原脚本。

## 功能

- 导入 `.xlsx` 或 `.csv` 表格。
- 自动扫描所有含链接的列，优先识别真实发布链接列。
- 可读取 `序号` 和 `昵称` 列。
- 支持视频号、头条号、抖音和小红书作品/笔记链接，包括 `https://weixin.qq.com/sph/...`、`https://channels.weixin.qq.com/...`、`https://m.toutiao.com/is/...`、`https://www.toutiao.com/...`、`https://v.douyin.com/...`、`https://www.douyin.com/video/...`、`https://www.douyin.com/note/...`、`https://www.xiaohongshu.com/explore/...`、`https://www.xiaohongshu.com/discovery/item/...` 与 `https://xhslink.com/...`。
- 使用真实 Chrome 标签页打开页面并截图。
- 图片下载到 Chrome 默认下载目录下的 `截图/<时间戳>_<源文件名>` 子文件夹，每次运行自动创建独立文件夹，不会混淆多次任务的截图。
- 文件名格式为 `序号_昵称.png`。
- 支持并发标签页、截图间隔、页面等待时间、暂停、继续、停止。
- 抖音任务会自动启用保守模式：单并发、前台加载、最长 60 秒作品页验证、任务间冷却 3 秒，并每 20 条重建截图窗口；可选通过 Clash 控制器同步轮换 VPN IP。
- 支持在插件内从截图 ZIP 或文件夹生成 PPT：`发布剪报多图铺页`、`链接截图单图单页` 和 `发布信息截图单图单页` 三种模式。
- 生成 `发布信息截图单图单页` PPT 时，可选调用 Reflo 插件 API 批量补全发布账号、发布标题、发布时间等发布信息；接口失败时会继续使用 Excel/CSV 字段。
- 支持通过“Excel 纠错 / Excel 修正”调用 Reflo 校验导入表中的发布日期、发布标题、发布平台、发布账号和链接；纠错导出带颜色标记的检查结果，修正会自动回填可由 Reflo 确认的字段。

## 安装方式

1. 打开 Chrome。
2. 进入 `chrome://extensions/`。
3. 打开右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择当前目录：

```text
shipinhao_chrome_extension
```

## 使用方式

1. 点击 Chrome 工具栏中的“视频号/头条号/抖音/小红书批量截图助手”。
2. 选择 Excel 或 CSV 文件。
3. 如需生成 `发布信息截图单图单页` PPT 或使用 Excel 纠错/修正，并使用更准确的发布信息，启用底部“发布信息增强”；首次使用、换电脑或 token 失效时，展开“内部配置”填写一次 Reflo 插件 token，token 只保存在本机 Chrome 存储，不写入插件源码。
4. 设置参数：
   - 并发标签页：默认 `2`，最多 `8`；并发用于页面预加载，实际截图会串行执行以避免错页。
   - 截图间隔秒：默认 `0.8`。
   - 页面等待秒：默认 `12`。
   - 抖音截图窗口：默认普通专用窗口；可选择无痕窗口，但需要在扩展详情中开启“允许在无痕模式下运行”。
   - 抖音每 20 条轮换 VPN IP：需要 Clash/VPN 客户端正在运行，默认控制器 `http://127.0.0.1:9090`、策略组 `Proxy`；如配置了 `secret`，请填入相同密钥；可填写指定轮换节点，出口 IP 未变化时会继续尝试下一个指定节点。
   - 截图画面：默认保留当前窗口 `1440×1200`；可选择手机竖屏 `9:16`、横屏 `16:9`、方图 `1:1` 或自定义宽高。
   - 最多处理条数：测试时可填写 `1` 或 `3`。
5. 确认识别到的任务数量和任务预览。
6. 点击“开始截图”。
7. 只有同时勾选“截图完成后同时生成当前 PPT 模式”且当前 PPT 模式为 `发布信息截图单图单页` 时，点击“开始截图”才会在截图前自动调用 Reflo；单独点击“生成并下载 PPT”且模式为 `发布信息截图单图单页` 时也会调用。其他截图或 PPT 模式不会调用。
8. 截图会自动下载到 Chrome 下载目录中的 `截图/<时间戳>_<源文件名>` 子文件夹，每次运行独立存放。
9. 默认勾选“截图完成后同时生成当前 PPT 模式”，插件会使用本次成功截图自动生成 PPT；也可以在“用截图生成 PPT”中选择已有截图 ZIP 或截图文件夹并点击“生成并下载 PPT”。
10. 如需检查导入表，点击底部“Excel 纠错”；如需让 Reflo 自动回填可确认的发布日期、发布标题、发布平台等字段，点击“Excel 修正”。红色表示链接无法获取发布信息，紫色表示重复链接，蓝色表示发布账号重复，黄色表示字段与 Reflo 返回值不一致；无法靠 Reflo 自动修复的问题仍会保留颜色和“纠错说明”供手动处理。

建议首次使用时先把“最多处理条数”设置为 `1` 或 `3` 做小批量验证，确认页面打开、截图内容和下载路径都正常后再全量运行。

如果处理抖音链接，请先在同一个 Chrome 浏览器中登录抖音；插件无法复用 Python 版本的 `auth.json`，短链会在真实标签页中等待跳转，不会在导入阶段联网解析。选择无痕窗口时，无痕环境通常不共享普通窗口登录态，需要先允许扩展在无痕模式下运行，并在无痕窗口登录抖音。

如果处理小红书链接，建议先在同一个 Chrome 浏览器中登录小红书。`xhslink.com` 短链会在截图窗口中等待跳转到小红书笔记页；如页面出现验证、空白或登录弹窗，请先小批量测试，并适当增加“页面等待秒”。

## 并发加速说明

插件采用“页面加载并发 + 截图限速”的方式：

- 多个 worker 可以并发打开和准备页面。
- 插件会创建专用截图窗口，并在每次截图前自动拉回前台、激活目标标签页、校验当前标签页后再截图。
- `captureVisibleTab` 截图调用会串行节流，避免触发 Chrome 截图频率限制，也避免把 A 页面保存成 B 文件名。
- 默认并发 `2` 比较稳妥；如果网络和电脑性能较好，可以尝试 `3` 到 `8`。
- 只要任务中包含抖音链接，插件会自动把并发降为 `1`，并按每 20 条抖音任务关闭、重建一次截图窗口；启用 VPN 轮换时，会在旧窗口关闭后、新窗口创建前切换 Clash 策略组节点。
- 如果出现白屏、页面加载不完整或截图窗口频繁被打断，优先把并发调低，并把页面等待秒增大。
- 截图画面尺寸会先创建/调整专用 Chrome 窗口，并在截图前校准网页可见区域 `window.innerWidth/innerHeight`，不是后期裁剪图片。
- `9:16` 适合模拟手机竖屏画面，`16:9` 适合横屏画面；如果需要精确尺寸，使用自定义宽高并先小批量验证。
- 普通 Chrome 插件无法真正后台截图或锁定窗口置顶；如果用户切到微信、飞书等窗口，插件会在下一次截图前尝试把专用截图窗口拉回前台。

## 和 Python 版本的差异

- 插件运行在真实 Chrome 中，更容易复用内部人员自己的浏览器环境；抖音登录态需要使用当前 Chrome 自身的登录状态。
- 插件不能真正重启整个 Chrome 进程；抖音批次重启会关闭并重建插件自己的截图窗口和标签页，以尽量模拟 Python 版本每批重新创建浏览器上下文的行为。
- 抖音无痕模式依赖 Chrome 扩展权限设置；如果未开启无痕权限，插件会回退普通截图窗口并在日志中提示。
- 插件保存文件受 Chrome 下载策略影响，不能像 Python 一样直接写入任意本地目录。
- 插件截图为专用 Chrome 窗口中的当前可见区域截图，不是后台截图，也不是 Playwright 的完整页面截图。
- 如果 Chrome 提示下载多个文件，请选择允许。
- 如果任务失败，插件会尝试在当前任务的 `截图/<时间戳>` 子文件夹中额外导出 `失败清单_时间.csv`。

## 插件内生成 PPT

插件内提供三种 PPT 模式：

- `发布剪报多图铺页`：使用内置 `发布剪报-模板(1)(1).pptx`，自动把多张截图铺到发布剪报页。
- `链接截图单图单页`：使用内置 `链接截图单图单页-模板.pptx`，每张截图生成一页；如果当前已导入 Excel/CSV 任务，会优先按截图文件名匹配任务的序号、昵称和链接，匹配不到时按自然排序顺序兜底，并用截图文件名补足缺失信息。
- `发布信息截图单图单页`：使用内置 `新单图单页-模板.pptx`，每张截图生成一页；左侧填充发布账号、发布平台、发布标题、发布链接、发布时间，右侧放截图，顶部大标题可在插件界面修改。字段优先使用 Reflo API 返回的 `account`、`platformLabel`、`title`、`publishTime` 等发布信息；Reflo 未启用、接口失败或单条失败时，回退读取 Excel/CSV 中表头包含 `账号` 或 `昵称` 的列、包含 `平台` 的列、包含 `标题` 的列，以及 `发布链接`/`链接`、`日期`/`发布日期`/`发布时间` 列。

三种模式都支持选择只包含截图图片的 ZIP，或直接选择截图文件夹。截图任务开始前如果勾选“截图完成后同时生成当前 PPT 模式”，完成后会直接使用本次成功截图生成 PPT，无需再次选择截图文件夹。生成后的 PPT 会下载到 Chrome 默认下载目录下的 `截图/<时间戳>_<源文件名>` 子文件夹。

## 本地生成发布剪报 PPT

如果截图图片较多、图片较大，建议使用本地 Python 生成器和插件版结果做对比。本地生成器读取只包含截图图片的 ZIP，并插入到内置 `发布剪报-模板(1)(1).pptx` 的“发布剪报”页中；会移除中间“一页贴15长图”占位框，贴图区域从标题下方约 5px 开始。

```bash
python3 tools/generate_clippings_pptx.py /path/to/screenshots.zip
```

指定输出文件：

```bash
python3 tools/generate_clippings_pptx.py /path/to/screenshots.zip -o /path/to/output.pptx
```

调整每页最多图片数量：

```bash
python3 tools/generate_clippings_pptx.py /path/to/screenshots.zip --max-per-slide 15
```

说明：

- 支持 `png`、`jpg`、`jpeg`；`webp` 需要安装 Pillow 后自动转为 PNG。
- ZIP 内图片会按自然文件名排序，例如 `1.png`、`2.png`、`10.png`。
- 多页时会尽量平均分配每页图片数量。
- 如果要处理 `webp`，先运行 `python3 -m pip install Pillow`。

本地结构验证：

```bash
python3 tools/verify_clippings_pptx.py
```

后续测试优先复用 `tools/` 下的脚本文件，避免在命令行内写大段临时代码。

## 表格要求

表格中建议包含以下列：

```text
序号 | 昵称 | 发布链接
```

插件会扫描所有含链接的列并优先选择视频号、头条号、抖音或小红书发布链接。`视频链接`、素材 MP4 链接、下载链接和不支持的第三方短链不会被当作截图任务链接；`v.douyin.com` 抖音短链和 `xhslink.com` 小红书短链会作为任务导入，并在截图窗口中等待跳转到作品/笔记页。
